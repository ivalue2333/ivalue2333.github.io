【数据结构与算法】 开篇

发表于 1970-01-01 更新于 2020-07-20 分类于 数据结构与算法
本文字数： 2k 阅读时长 ≈ 2 分钟

模型

今天也是有了一点新的感悟。所谓的框架，套路，原则，他们很多时候在讲的事情或者说理念很接近。但是今天我感觉可以用一个更好的词语来描述这种思考的方式，模型。你看，解决一个数学问题，你可以用特定一个或几个的模型去解决；解决一道算法题，它也会被归纳为dfs， dp， double pointer等模型；甚至一个架构上的问题，也可以套用许多先用的模型拼凑出适合的模型。最终，核心的难点是两个，如何高效的学习模型，如何灵活地运用模型。

迷思

前前后后做了100来道算法题，但是在算法方面总是感觉问题很大。一直在思考如何解决这个问题。后面逐渐明白需要将算法题分类，其实分类后就是那么几类，每一种类型都有个大概的解法。可是还是不够。核心的问题是没有系统性的总结，没有形成可复用的系统性的框架，一些模模糊糊的总结实际上不能帮助我在遇到新的问题的时候，有非常敏捷的反应来处理对应的问题。直到看到了算法小抄，总体的感觉是他把我想做的事情做得很好（我想很多人也有这种经历吧，别人将自己想做的事做了出来，而且做得更完善，更易用，更被接受）。

不过，站在别人的肩膀上，总能走得更远，找到了更为科学的道路，那么剩下的事情就是将这条路走通就可以了。

还是有几点可以总结的。

  • 站在别人的肩膀上：其实做一件事最好的方式，不是自己想出解决办法，而是用别人实践出来的已经成功的方法。这就需要足够广泛地搜索已经成功的方法，并选择或者综合其中的几种，使其成为自己的方法，这应该是学习的极佳的方法吧。
    • 向成功的人学习
  • 足够的抽象（或者叫套路）：当一个模式需要广泛的使用时，需要像病毒一样的复制时，他需要足够的抽象，抽象式的框架便可以确保足够的复用度。
  • 不要盲目拔高：感觉我之前就有不好的心理，一说算法，那肯定就是要看《算法之美》。盲目的拔高就意味着没有足够的基础，就像是让一个小学生学习微分方程一样，不是该干那个事情的时候。所以，书是一页一页的看的，能力是一点一点地提高的。当你有了基础，那么再去拔高才是合适的时候。

再论如何学习

我，或许很多其他的人也是一样，认为知道一个问题的解法不重要，一定要自己想出来才是最厉害的。这是一种典型的错误（当然这句话不一定全对）。不管是高中的数学物理，还是大学的高数离散，还是程序中的算法，其实都是一样的。当一个问题作为题来用时，首先不应该是自己去想如何解决，首先应该看以前的人是如何解决这个问题，当然问题和解答方法肯定会被分类然后形成所谓的套路。当你站在了以前的人的肩膀上时（这是在极其有限的时间里极其有效率的方法），你才能更系统，更全面，最重要的，最快速的掌握这些问题的解决方法，并以这些方法和框架为基础，解决更多的问题。

再论学习资料

大三下班学期的时候，我就被学习资料给坑了。当时是什么呢，当时是希望走Java开发的路线，然后顺着经典的学习思路，找来了几本 Java 书籍就在那看。什么 jsp， java桌面系统看了一堆。最终发现我在这些书籍中学习到的知识是如此的陈旧（软件行业发展和迭代太快也是一方面）以至于我学的这些东西，市面上都没有公司在用了，都在 spring，更别说谁还在用 java 写桌面客户端啊（jetbrain…）。现在回想起来，当时如果及时调整学习资料和学习策略，或许结果大不一样。

另一个方面，就目前来说，我在网络上找到了许多 算法和数据结构啊， spring boot啊，这些非常好的学习资料。但是！当我想深入研究一下 InnoDB 时，我又发现，我还是老老实实去啃书是最好的。因为当我开始深入研究时，网络上那些高度总结概括的文章会把许多重要细节隐藏掉。

结论就是，优先看别人怎么学，从哪里学，从网上和书中去寻找答案。顺便说一下，10 年前的书，在软件行业，基本上也没看的必要了（个别经典除外）。最后，多向他人学习，并独立思考

从数据分析到再论思考的模型

数据分析中有三个非常基本的概念，分别是结构化，公式化，业务化。所谓结构化，简单来说就是一层一层的分类。公式化，简单来说，就是将你分析的数据用公式来表示，比如，直播的销售额 = 客单件 * 成交人数（当然还可以继续分），所谓的业务化，就是你不能只是讨论一些数据分析的方法，然后结合一些公式就说你分析完了，你还要结合具体的业务来分析。这也适用于其他问题的思考

  • 公式化
    • 分类，mece 原则，穷尽性与互斥性
  • 公式化
    • 通过公式列出影响因子
  • 业务化
    • 结合业务做精准分析

mece 分析法

MECE，是Mutually Exclusive Collectively Exhaustive，中文意思是“相互独立，完全穷尽”。 也就是对于一个重大的议题，能够做到不重叠、不遗漏的分类，而且能够借此有效把握问题的核心，并解决问题的方法。独立性和穷尽性

算法

算法的入门是什么，其实很简单，就是你就按 if + while 来穷举，穷举法是所有的算法思想的基础，在这之上的各种算法都是在想办法优化时间复杂度。
