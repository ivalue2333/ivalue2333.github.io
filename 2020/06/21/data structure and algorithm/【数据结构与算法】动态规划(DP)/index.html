算法框架

  1. 找子问题
  2. 动态规划的核心是状态转换方程
  3. 要写出状态转换方程，就需要先写出状态方程
  4. 状态方程的自变量肯定是状态，因变量一般来说是要求的极值, y = f(i)
  5. 状态就是问题和子问题中的变量（自变量），一般来说都是[0, n)（一维数组）
  6. 状态之所以变化是因为做了选择，在每一个状态都有一个选择列表
  7. 回到第1点，问题一般是多个子问题中的极值（因为要求的就是极值）。问题和子问题的区别一般是数组的长度或字符串的长度有区别（这就是状态，或者说自变量）。
  8. 所以定义子问题很关键

状态转换方程就是，子问题和问题的关系。由于在一种状态下，可做的选择是有限的，所以对应的子问题个数也是有限的，所以最优子问题（子结构）是可以确定，而问题一般就是等于最优子问题（加上一个常数）

coin change

  1. 确定「状态」，也就是原问题和子问题中会变化的变量。由于硬币数量无限，硬币的面额也是题目给定的，只有目标金额会不断地向 base case 靠近，所以唯一的「状态」就是目标金额 amount。
  2. 确定「选择」，也就是导致「状态」产生变化的行为。目标金额为什么变化呢，因为你在选择硬币，你每选择一枚硬币，就相当于减少了目标金额。所以说所有硬币的面值，就是你的「选择」。
  3. 确定 [基础状态]，当状态对应的值可直接硬编码得出时的状态。
  4. 明确 dp 函数/数组（使用函数一般是自顶向下的递归，使用数组一般是自底向上的递推） 的定义。我们这里讲的是自顶向下的解法，所以会有一个递归的 dp 函数，一般来说函数的参数就是状态转移中会变化的量，也就是上面说到的「状态」；函数的返回值就是题目要求我们计算的量。

所以dp(n) 的定义：输入一个目标金额 n，返回凑出目标金额 n 的最少硬币数量。 即 dp[状态] 映射 计算量。
