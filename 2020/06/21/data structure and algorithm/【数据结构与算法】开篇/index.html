迷思

前前后后做了100来道算法题，但是在算法方面总是感觉问题很大。一直在思考如何解决这个问题。后面逐渐明白需要将算法题分类，其实分类后就是那么几类，每一种类型都有个大概的解法。可是还是不够。核心的问题是没有系统性的总结，没有形成可复用的系统性的框架，一些模模糊糊的总结实际上不能帮助我在遇到新的问题的时候，有非常敏捷的反应来处理对应的问题。直到看到了算法小抄，总体的感觉是他把我想做的事情做得很好（我想很多人也有这种经历吧，别人将自己想做的事做了出来，而且做得更完善，更易用，更被接受）。

不过，站在别人的肩膀上，总能走得更远，找到了更为科学的道路，那么剩下的事情就是将这条路走通就可以了。

还是有几点可以总结的。

  • 站在别人的肩膀上：其实做一件事最好的方式，不是自己想出解决办法，而是用别人实践出来的已经成功的方法。这就需要足够广泛地搜索已经成功的方法，并选择或者综合其中的几种，使其成为自己的方法，这应该是学习的极佳的方法吧。
    • 向成功的人学习
  • 足够的抽象（或者叫套路）：当一个模式需要广泛的使用时，需要像病毒一样的复制时，他需要足够的抽象，抽象式的框架便可以确保足够的复用度。
  • 不要盲目拔高：感觉我之前就有不好的心理，一说算法，那肯定就是要看《算法之美》。盲目的拔高就意味着没有足够的基础，就像是让一个小学生学习微分方程一样，不是该干那个事情的时候。所以，书是一页一页的看的，能力是一点一点地提高的。当你有了基础，那么再去拔高才是合适的时候。

再论如何学习

我，或许很多其他的人也是一样，认为知道一个问题的解法不重要，一定要自己想出来才是最厉害的。这是一种典型的错误（当然这句话不一定全对）。不管是高中的数学物理，还是大学的高数离散，还是程序中的算法，其实都是一样的。当一个问题作为题来用时，首先不应该是自己去想如何解决，首先应该看以前的人是如何解决这个问题，当然问题和解答方法肯定会被分类然后形成所谓的套路。当你站在了以前的人的肩膀上时（这是在极其有限的时间里极其有效率的方法），你才能更系统，更全面，最重要的，最快速的掌握这些问题的解决方法，并以这些方法和框架为基础，解决更多的问题。

算法

算法的入门是什么，其实很简单，就是你就按 if + while 来穷举，穷举法是所有的算法思想的基础，在这之上的各种算法都是在想办法优化时间复杂度。
